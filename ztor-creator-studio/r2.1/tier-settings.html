Rules & versioning

How the tier system behaves. These are product rules — surfaced here, owned upstream.

Dual-gate upgrade
A fan upgrades only when reputation AND spend both pass the tier’s gates (§7.5).
Monotonic thresholds
Higher tiers must require strictly more than lower tiers, or the save is blocked. Exact validation — TBC, not invented here.
Versioned, non-retroactive
Each save creates an immutable tier config version. Past revenue and badges earned under an old version are not rewritten (mirrors §7.6 rate versioning). Versioning behaviour — TBC, owned upstream.
When changes take effect
Threshold and multiplier changes apply at the next nightly recompute. Benefit changes apply immediately.
