Creator-economy operations dashboard.創作者經濟營運儀表板

The back-office where independent creators turn IP, projects, and fans into income — transparently. Clean white canvas with neutral light-gray surfaces, highlighter-orange primary, Geist typography, multi-layer rim+drop shadows in place of borders.

Category — Creator-economy operations dashboard: IP licensing, projects, events, products, fans (CRM), and earnings, with finance / royalties / verification states kept legible. Theme: light / dark / system.

Discipline

Top-level Do / Don't.最高原則 該做 / 別做

Do該做

  • Reserve #ffa33f for one structural accent per viewport (CTA, active nav, KPI highlight, sticky-note, or hero — not several at once).
  • 每個視窗只保留 #ffa33f 給一個結構性重點（CTA、active nav、KPI highlight、sticky-note 或 hero，不要同時用好幾個）。
  • Pair --primary with --primary-foreground (#171717) text — never white-on-orange.
  • --primary 一律搭配 --primary-foreground（#171717）文字，絕不用白字配橘底。
  • Define edges with multi-layer shadows (--shadow-card = rim + drop), not 1px borders.
  • 用多層陰影（--shadow-card = rim + drop）來定義邊緣，而非 1px 邊框。
  • Spell out money state inline: "Available $842 · Pending $1,204".
  • 把金額狀態直接寫出來：「Available $842 · Pending $1,204」。

Don't別做

  • Don't use orange for nav active states' fill or status pills — those use --muted / status tokens.
  • 別把橘色用在 nav active 狀態的填色或 status pill，那些要用 --muted / status token。
  • Don't draw a 1px solid border on cards / inputs — the shadow/hairline is the edge.
  • 別在 cards / inputs 上畫 1px solid 邊框，陰影 / hairline 就是邊緣。
  • Don't show a bare number without its currency + state.
  • 別只顯示沒有幣別與狀態的裸數字。
  • Don't use Title Case on buttons — sentence case ("Create project").
  • 按鈕別用 Title Case，用 sentence case（「Create project」）。
  • Don't put more than one Primary button in a row.
  • 同一列別放超過一個 Primary 按鈕。

2.6 Cross-component rules

2.6 跨元件規則

These are principles every component obeys, not a token scale — rendered as live mini-demos.

這些是每個元件都要遵守的原則，不是 token 刻度——用小型 live 示例呈現。

① Focus 環

One recipe across every control + both modes — outline: 2px solid var(--ring); outline-offset: 2px (list rows use -2px inset). No per-component outline vs box-shadow drift.

全控件、兩模式單一配方 —— outline: 2px solid var(--ring); outline-offset: 2px（清單列用 -2px 內嵌）。不再各元件各寫各的。

② Surface 分層對比

A component is seen by the contrast of its fill / border / shadow against the layer it sits on; a fill the same colour as the background disappears. White fills need a 1px border to form on white; for cross-layer safety use a real border, not fill-as-edge. Build any filled component against its layers and verify on the darkest one. (Button white/grey example: §4.2.)

元件靠「填色／邊框／陰影跟所在那層的對比」被看見，填色和背景同色就會消失。白填要靠 1px 邊框在白底成形；要跨層安全就用實線 border，別用「填色當邊」。做任何有填色的元件，對它所在的層做、並在最深那層目視驗證。（按鈕白/灰底實例見 §4.2。）

white on white · no border → vanishes白填白底 · 無邊 → 消失
+ 1px border → forms＋1px 邊 → 成形

③ Primary 保留用途

--primary (orange) is only for primary CTA + sticky-note + brand mark + hero accent. Never nav active, KPI highlight, or status pills — those use --muted/--accent.

4.2 Button

🟢 atom

Purpose

用途

Primary action control. Five emphasis levels — pick by how important the action is and how dense the surface is. One orange primary per block (card / dialog / toolbar); never use the orange primary for destructive actions.

主要動作控制項。五種強度——依「動作多重要」與「版面多密」挑。每一塊（卡片／彈窗／工具列）只留一個橘 primary;刪除類動作絕不用橘色 primary。

When to use which

什麼時候用哪個

“One block” = a self-contained area on screen (a card, a dialog / form, a toolbar, a list row) — not the whole page.

「一塊」＝畫面上各自獨立的一塊區域（一張卡片、一個彈窗／表單、工具列、列表的一列），不是整頁。

Outline vs Soft＝要不要框：白內容區要邊界 → outline；工具列／密集區要安靜 → soft。目前無紅色 destructive 變體——需要時補 .btn--danger;刪除別用橘 primary。

On which background (white vs grey)

配哪種底色（白底 vs 灰底）

A button is seen by the contrast of its fill/border against the surface — a fill the same colour as the background disappears. So pick by the surface layer it sits on (ramp: white --card → grey --muted/--surface-shell/--sidebar).

按鈕靠「填色／邊框跟背景的對比」被看見——填色和背景同色就會消失。所以要看它坐在哪一層 surface（階梯：白 --card → 灰 --muted/--surface-shell/--sidebar）。

In short: soft is white-surface only; on grey use outline. Outline has a real border, so it holds on both — the safest cross-layer secondary.

一句話：soft 只活在白底；要放灰底就改 outline。 outline 有實線框，白灰底都成立，是跨層最安全的次要。
