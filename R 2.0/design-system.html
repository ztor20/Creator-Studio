AI search analytics — with a louder voice.

01 · Foundations

Color, type, geometry.

Paper-tone monochrome canvas with status accents that only appear as data dots inside dashboards. Subtle 6–7px radii, multi-layer shadows in place of borders, and Geist as the structural font with Inter reserved for paragraph copy.

1.1 Color

— 4 surfaces / 3 text / 4 status

Single high-saturation brand color: highlighter yellow #FFDB29 plays primary CTA and nothing else does. Canvas warms slightly (#FAFAF7) and hairlines warm to match (#D1D1C7) so the yellow doesn't sit on a cold gray. Status hues stay reserved for data dots inside dashboard demos. ⚡ = changed from ztor.

1.4 Radius

— fine-grained, 6–8px dominant

Ztor's radius system is fine-grained subtle at the chrome layer — 6–8px dominates — with a clear leap to full pills for round elements. App-level editorial cards escalate to 12 / 16 / 24px for quote and verdict cards. Visual opposite of Midday's 0px brand decision.

1.5 Shadow

— multi-layer rim+drop replaces border

Ztor uses multi-layer shadows to define edges without drawing a border. The inset 0 0 0 1px rgba() ring is a soft outline; the 0 2px 6px rgba() is the drop. Together they replace what most systems would draw as border: 1px solid var(--border).

Ztor uses Framer's default transition stack: ~200ms duration with cubic-bezier(0.32, 0.72, 0, 1) ease-out-quart curve. App-side dialog mounts run at exactly 200ms with scale 0.95 → 1 + opacity 0 → 1.
